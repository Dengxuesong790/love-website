## 💻 本地测试

### 方法一：直接打开
双击 `index.html` 文件即可在浏览器中打开预览。

### 方法二：使用本地服务器（推荐）

#### VS Code Live Server
1. 安装 VS Code 的 Live Server 插件
2. 右键点击 index.html
3. 选择 "Open with Live Server"

#### Python 简易服务器
```bash
# Python 3
python -m http.server 8000

# Python 2
python -m SimpleHTTPServer 8000
```
然后访问 http://localhost:8000

#### Node.js http-server
```bash
npx http-server -p 8000
```
然后访问 http://localhost:8000

## 🌐 免费托管平台部署

### GitHub Pages（推荐⭐）

#### 步骤：
1. 创建 GitHub 账号（如果没有）
2. 创建一个新的仓库，例如 `love-website`
3. 将所有文件上传到仓库
4. 进入 Settings > Pages
5. Source 选择 `main branch`
6. 等待几分钟，网站就会发布
7. 访问地址：`https://你的用户名.github.io/仓库名`

#### 优点：
- ✅ 完全免费
- ✅ 自动 HTTPS
- ✅ 支持自定义域名
- ✅ 版本管理方便

---

### Vercel

#### 步骤：
1. 访问 https://vercel.com
2. 使用 GitHub 账号登录
3. 点击 "New Project"
4. 导入你的 GitHub 仓库
5. 点击 "Deploy"
6. 等待部署完成

#### 优点：
- ✅ 自动部署
- ✅ 全球 CDN
- ✅ 性能优秀
- ✅ 支持自定义域名

---

### Netlify

#### 步骤：
1. 访问 https://netlify.com
2. 注册账号
3. 拖拽整个项目文件夹到部署区域
4. 或者连接 GitHub 仓库自动部署

#### 优点：
- ✅ 拖拽即可部署
- ✅ 自动 HTTPS
- ✅ 表单收集功能
- ✅ 支持密码保护

---

## 🔧 自定义域名（可选）

### 购买域名
- 国内：阿里云、腾讯云、华为云
- 国外：Namecheap、GoDaddy、Google Domains

### 绑定域名步骤（以 GitHub Pages 为例）

1. 在仓库根目录创建 `CNAME` 文件
2. 文件中写入你的域名，例如：
   ```
   love.yourdomain.com
   ```
3. 提交更改
4. 到你的域名服务商添加 CNAME 记录
5. 等待 DNS 生效（通常几分钟到几小时）

## ⚡ 性能优化建议

### 图片优化
- 使用 TinyPNG 等工具压缩图片
- 推荐使用 WebP 格式（更小的体积）
- 保持图片质量的同时减小文件大小

### 代码优化
- CSS 和 JS 文件已优化，无需额外处理
- 如需进一步压缩，可使用在线工具

### 加载优化
- 图片已实现懒加载
- 音乐文件不要超过 5MB
- 避免使用过大的背景图片

## 🆘 常见问题

### Q: 部署后图片不显示？
A: 检查图片路径是否正确，文件名是否匹配。

### Q: 音乐无法播放？
A: 现代浏览器需要用户交互后才能播放音频，点击音乐按钮即可。

### Q: 手机端样式错乱？
A: 清除浏览器缓存，或检查 viewport 设置。

### Q: 如何更新网站内容？
A: 修改相应文件后重新上传或推送代码到仓库。

### Q: 可以多人协作编辑吗？
A: 可以，使用 Git 进行版本管理和协作。
